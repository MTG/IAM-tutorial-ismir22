Pitch extraction

Pitch extraction#

As seen in the melodic introduction, predominant and vocal pitch is a very relevant feature to tackle the melodic analysis of Carnatic and Hindustani Music.

%pip install compiam

## Importing compiam to the project
import compiam

# Import extras and supress warnings to keep the tutorial clean
from pprint import pprint
import warnings
warnings.filterwarnings('ignore')

pprint("Available methods for pitch extraction: {}".\
    format(compiam.melody.pitch_extraction.list_tools()))

"Available methods for pitch extraction: ['FTANetCarnatic*', 'Melodia']"

In an Indian Art Music context, this task has been mainly approached through heuristic-based approaches [RR10, SG12], which have been used yet in recent years.

Let’s extract the pitch from an audio sample using Melodia [SG12]. We first need to install essentia, which is the optional dependency required to load this tool.

%pip install essentia

from compiam.melody.pitch_extraction import Melodia
melodia = Melodia()  # initializing a melodia instance
pitch_track = melodia.extract("../audio/testing_samples/test_1.wav")

[   INFO   ] MusicExtractorSVM: no classifier models were configured by default

Melodia has been found, in the original paper experiments and also in the MIREX campaign, to decently work on Indian Art Music samples. However, recent DL-based models have claimed the state-of-the-art for the task of pitch extraction. Maybe we can use a Carnatic-trained version of one of these models to extract the pitch?

Let us first uninstall essentia to clean our environment from optional dependencies, and start from scratch for the next tool.

%pip uninstall -y essentia

And let’s now import a DL model that learns to automatically extract the predominant melody from audio recordings.

from compiam.melody.pitch_extraction import FTANetCarnatic

This tool requires an optional dependency, as seen in the error message that was thrown. Therefore we first need to install the required dependency and load the tool again.

%pip install tensorflow==2.7.2

Let’s first deactivate the GPU usage, since we assume no CUDA-capable GPU is available in most of the cases. We import tensorflow and set the visible GPU devices to none.
